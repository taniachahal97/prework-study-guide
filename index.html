HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • The h1, h2, h3, h4, h5, h6 tags represent the level of heading a given text block represents
  • The p element represents a paragraph or block of text
  • The ul, ol, and li elements represent unordered lists,, ordered lists and list items
  • The img element contains information about images that are displayed on the webpage

CSS

  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element

Git

  • The git workflow is as follows: Create a Branch, Open A Pull Request, Merge and Deploy
  • Create a new branch from the most recent version of the main branch where entire team works
  • Commit and push your work to the remote new branch
  • Open a pull request and assign team mates to review your content changes and approve your pull request
  • Merge the approved changes into the main branch, preview on site, and then publish
  • git status:checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it

JavaScript

  • A variable is a named container that allows us to store data in our code
  • Control flow is the order in which a computer executes code in a script
  • Typically, JavaScript code is executed from top to bottom
  • Using conditional statements allows us to interrupt the control flow
